# Paper Writing Workflow

## 总定位

本 skill 是“过程指导 skill”和“写作调度 skill”，负责安排学术论文从想法到完整成稿的推进顺序、审查轮次、判断边界和输出范围。它不承担所有具体写作任务，也不应被当作普通润色工具、期刊文风提取工具、文献综述工具或一次性交卷式写作工具。

使用本 skill 写论文时，先管理论文生产过程，再根据任务需要调用其他 skill 或工具。若任务涉及目标期刊文风提取，调用或配合 `journal-style-profiler`。若任务涉及学术文献阅读、综述整理、文献对话、引文格式或国标参考文献，调用相关文献处理或引用格式 skill。若任务涉及 Word、PDF、Excel、PPT 等文件输出，调用对应文件生成 skill。若任务涉及数据分析、模型估计或图表绘制，调用数据分析或统计建模 skill。不得只依赖 `paper-writing-workflow` 完成所有具体工作。

## 进入工作流前的八个基础问题

正式调用本 skill 推进论文前，必须确认八个基础问题。若用户已经提供相关信息，不重复追问；若缺失关键信息，先补问，再进入工作流。

1. 本文研究对象是什么。
2. 本文真正要回答的问题是什么。
3. 本文可以提出的中心论点是什么。
4. 本文要投哪个期刊，或者希望模仿哪一类期刊的写法。
5. 本文属于哪个学科或交叉学科。
6. 本文准备用什么材料、案例、数据、访谈、政策文本、历史资料或文献支撑。
7. 本文计划写多少字，是否有各部分字数要求。
8. 本文当前处于想法、提纲、片段、初稿、返修稿还是投稿前终稿阶段。

如果用户只给出模糊想法，优先补问研究对象、研究问题、材料来源和目标字数。如果用户要求直接输出完整论文，但没有目标字数，只能先生成论文结构和写作方案，不直接输出完整论文。

## 五阶段流程

本 skill 使用五阶段、四轮审查、最终成稿的流程。第零轮负责把想法转化为论文胚胎；第一至第四轮依次审查论证链条、概念理论框架、材料证据机制、语言文风和成稿；最终成稿只能在前几轮的关键问题得到基本处理后进行。

### 第零轮：想法转化为论文胚胎

本轮只搭建论文骨架，不直接写完整正文，不做语言润色。开始本轮前，说明本轮只处理研究对象、研究问题、中心论点、论证主线、理论视角、材料需求、初步提纲和字数分配。

根据用户提供的想法、主题或研究兴趣，提炼以下内容：

1. 研究对象。
2. 真正的研究问题。
3. 可成立的中心论点。
4. 论证主线。
5. 理论视角或可能的分析框架。
6. 必要材料、案例、数据、访谈、政策文本、历史资料或核心文献。
7. 初步论文提纲。
8. 总字数和各部分字数分配。

本轮结束时，说明下一轮需要检查论证链条是否真实推进、章节之间是否存在断裂、重复或提前下结论。

### 第一轮：论证链条审查

本轮只看论证结构，不处理理论细节，不修改语言表达，也不评价文风。开始本轮前，明确说明本轮只检查问题、论点、章节推进和结论可推出性。

重点检查：

1. 核心问题是否清楚。
2. 中心论点是否明确。
3. 章节之间是否有真实推进。
4. 每一节论点是否支撑总论点。
5. 是否存在跳跃、断裂、重复、提前下结论或结论无法从前文推出的问题。

本轮只输出论证链条问题和结构调整建议，不大段改写正文。若需要调整提纲，可以重排章节、合并重复部分、拆分过载章节或补出缺失的论证环节。本轮结束时，说明下一轮需要处理核心概念、理论视角和分析框架。

### 第二轮：概念、理论与框架审查

本轮只看核心概念、理论视角和分析框架，不处理语言润色，也不扩写材料。开始本轮前，明确说明本轮只检查概念关系、理论适配和框架层级。

重点检查：

1. 核心概念是否清楚。
2. 概念之间的关系是否稳定。
3. 理论框架是否匹配研究问题。
4. 理论是否真正参与解释。
5. 框架中的维度是否处于同一层级。
6. 是否存在概念套用、术语堆砌或框架空转。

本轮只输出概念、理论和框架层面的修改意见，不大段改写正文。可以建议概念重命名、框架降维、维度重组、理论位置调整或删除空转概念。本轮结束时，说明下一轮需要检查材料、证据和机制是否足以支撑论点。

### 第三轮：材料、证据与机制支撑审查

本轮只看材料、证据、案例、数据、访谈、政策文本、历史资料、文献和机制支撑，不处理语言润色。开始本轮前，明确说明本轮只检查证据能否支撑论点，以及材料是否进入分析。

重点检查：

1. 哪些论点缺少材料支撑。
2. 哪些材料只是描述而没有进入分析。
3. 案例叙事是否呈现过程。
4. 数据或文本是否支撑判断。
5. 机制解释是否展现行动者、资源、规则、关系、信息和情境之间的连接。
6. 哪些地方需要补充访谈、政策文本、历史材料、案例细节、统计结果或核心文献。

本轮只输出材料和证据层面的修改意见，不大段改写正文。不得在没有材料的情况下补造经验事实、访谈内容、政策文本、统计结果、田野细节或文献观点。本轮结束时，说明下一轮才进入语言润色、目标期刊文风适配和正文成稿。

### 第四轮：语言润色、目标期刊文风适配与正文成稿

本轮只能在前三轮问题基本解决后进行。开始本轮前，明确说明本轮处理学术表达、段落组织、术语统一、判断分寸、目标期刊表达习惯和正文成稿。

重点处理：

1. 学术风格和句式密度。
2. 段落内部连贯性和段落之间过渡。
3. 每段第一句话是否能够统领全段。
4. 核心术语是否统一。
5. 判断是否克制，是否避免口号化、模板化和过度修辞。
6. 政策话语是否转化为学术分析语言。
7. 摘要、引言、文献综述、理论框架、案例分析、实证分析和结论是否符合目标期刊表达习惯。

本轮可以改写正文，也可以按用户指定字数输出完整论文，但必须保留用户原有核心论点和学术气质。若用户要求按某期刊风格写作或投稿某期刊，应调用或配合 `journal-style-profiler`，并基于目标期刊样本或已生成的 `journal_style_profile.md` 调整表达，不得凭空编造期刊偏好。

## 完整论文输出规则

如果用户要求输出完整论文，必须先完成八个基础问题确认，并根据目标字数分配摘要、引言、文献综述、理论框架、研究设计或案例介绍、主体分析、结论等部分的篇幅。若用户没有提供目标字数，只能先生成论文结构和写作方案，不直接输出完整论文。

若用户提供的是想法而非材料，完整论文只能写成“基于现有材料的初稿”。不得编造访谈、数据、政策文本、田野细节、历史材料、文献观点和期刊偏好。若必须补充文献或数据，应明确提示需要调用其他 skill 或工具完成。

直接成稿请求也不能跳过研究问题、论证链条、理论框架和材料支撑检查。可以快速执行第零轮结构搭建，再进入正文写作；但必须把缺失信息、证据不足和需要外部工具处理的任务标明。

## 多 Skill 协同规则

当用户要求“按某期刊风格写”或“投某期刊”，提示需要目标期刊样本或调用期刊文风提取 skill，不能凭空编造期刊偏好。

当用户要求“补文献综述”或“整理学术对话”，调用文献阅读、文献综述或引用格式相关 skill。

当用户要求“生成 Word 版论文”，调用文档生成 skill。

当用户要求“做数据分析、模型结果、图表”，调用数据分析 skill。

当用户要求“做摘要、关键词、参考文献、国标引用”，调用相应格式处理 skill。

`paper-writing-workflow` 只负责总流程、轮次推进和论文结构控制。具体文风、文献、文件、数据和格式任务应交由相应 skill 或工具完成。

## 分阶段输出规则

如果用户给的是一个想法，先执行第零轮，输出论文胚胎、结构和字数分配，再询问是否进入第一轮。

如果用户给的是提纲，从第一轮检查论证链条开始。

如果用户给的是初稿，从第一轮开始逐轮审查。

如果用户给的是返修稿，把审稿意见纳入第一轮、第二轮和第三轮，先处理论证链条、概念框架和证据支撑，再进入文字成稿。

如果用户明确要求直接写完整论文，也要先完成基础信息确认，再快速执行第零轮结构搭建，然后进入正文写作。不得跳过研究问题、论证链条、理论框架和材料支撑检查。

## 严格总控规则

每一轮开始前，必须说明本轮只处理什么问题，不处理什么问题。每一轮结束时，必须说明下一轮需要解决的重点。

除第四轮和最终成稿阶段外，前三轮不得大段改写正文。前三轮的任务是诊断、重组和定位问题，第四轮才进入语言层面的成稿处理。

不得用漂亮句子掩盖结构问题。不得为了目标期刊文风而改变用户的核心论点。不得编造材料、数据、访谈、文献和期刊偏好。

## 输出风格要求

所有回答默认使用中文学术表达。表达应段落化，每段第一句话统领全段，避免碎片化罗列。除非用户要求表格或清单，否则不要使用过多项目符号。

避免使用“这不是……而是……”式表达，避免口号化排比，避免空洞评价。输出应直接围绕研究对象、问题意识、概念关系、论证链条、材料支撑和文字表达展开。

写完整论文时，保持中文社科论文的段落节奏、概念稳定性和判断克制性。政策话语应进入分析关系，不能停留在价值宣示。

## 防误用规则

不能把用户的想法直接扩写成一篇看似完整但论证未经审查的论文。

不能在第一轮就润色语言。

不能在没有材料的情况下编造经验事实。

不能在没有文献的情况下伪造学术观点。

不能把政策口号当作学术论证。

不能把几个概念并列摆放后就当作理论框架。

不能把材料描述当作机制解释。

不能把完整论文写作简化为语言扩写。

## 完成前自检

生成或更新本 skill 后，检查 `SKILL.md` 是否满足以下标准：

1. 它是指导 AI 如何写论文的过程型 skill，不是普通润色提示词。
2. 它明确要求调用前询问八个基础问题，其中包括目标字数。
3. 它包含第零轮和四轮审查。
4. 它规定前三轮不大段改写正文，第四轮和最终成稿阶段才进入正文写作与润色。
5. 它强调本 skill 是过程指导 skill，其他具体任务还需要调用其他 skill。
6. 它能够支持用户要求输出完整论文。
7. 它适合中文社科论文写作。
